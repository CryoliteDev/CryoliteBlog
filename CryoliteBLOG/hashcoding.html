Encoding Vs. Hashing Vs. Encryption

With widespread adoption of cloud computing, keeping sensitive information secure from theft and vulnerabilities can be a real challenging. Encoding, hashing and encrypting are common concepts when it comes to securing data.

Encoding

To encode something is to write in a way that other person/application will understand. There are many encoding standards such as ASCII, UTF-8 and Base64 just to name a few of them. A easy way to understand is to compare it to human language. The words and rules of each language are unique to each language. For example: An English speaker might not understand Chinese but with a translator he/she can easily decode the Chinese to English to understand the encoded text.

These might look unrelated to human eyes but to certain application they are the same thing. Relying on encoding to secure data is not a good practice that is because if anyone intercepts this data will have the real data since decoding these standards is an easy job.

Hashing

Hashing is an integrity validation method. Hash coding can indicate whether the data has been altered in transit because its original hash value and hash value computed by the receiver will not match after the data was altered hence it is not a way to secure data. 

For example, when downloading an .iso for a Linux distribution from online, you want to make sure that the .iso file you installed has not been altered in anyway so you would make sure that the .iso you are installing is original.

Common use of hashing is to secure the storage of passwords. When a password is created and then put through a hashing algorithm. The resulting hash is then stored. When user attempts to authenticate the password is hashed again and checked against the original hash to make sure it is the same.

Hashing algorithms include MD5, SHA1 and SHA256 are some of Hashing methods.

Hashing is never meant to be reversed so it should not be used to secure communications, it is just to check the integrity of data. It is not a encryption.

Encryption

Encryption is making data unreadable by anyone except those who have the decryption key. The decryption can be a single shared key, or it can be broken int two keys, one kept private and one given to the other party. The logic behind and encryption program that includes the mathematics used in to do the conversion from plain text to the encrypted text (cipher text) is known as a encryption algorithm.

Data such as messages are encrypted just before it is sent over a network or the internet and when the data arrives it is decrypted using the decryption program on device.

Some encryption algorithms are considered so important that the U.S. government has banned publication of details about them. Currently, it is illegal for U.S. companies to export some of these encryption algorithms. The Freedom Forum LINK contains a number of articles on lawsuits and legislation surrounding encryption export laws. Some consider these publication restrictions a freedom of speech issue.

One main property of an encryption algorithm is that someone can know the detail of the algorithm and still not be able to decipher the data without knowing the key that was used to encrypt the data. Most security experts today believe that a 128-bit key is decent security for data transmission, but now 256-bit and 2048-bit are commonly used now a days.

Understanding the difference between these concepts can help you with considering security design and architecture. Accidentally implementing encoding as an encryption method can be very dangerous to an organization as that would mean the transmissions are real data that can be easily decoded by anyone.
